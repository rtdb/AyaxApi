1. DEPLOY
Открыть файл /AyaxApi/AyaxApiSettings.json и указать имя машины на которой установлен MS SQL Server в Data Source="LAPTOP-R5MJF817".
Открыть файл /TestAyaxApi/AyaxApi.Tests.json и указать имя машины на которой установлен MS SQL Server в параметре Data Source="LAPTOP-R5MJF817".
При желании можно изменить имя БД в параметре Database="AyaxApiSample".

2. Комментарии к выполнению задания
Ввиду отсутсвия в приложении бизнеслогики (только CRUD) выбрана простая двухслойная архитектура приложения: 
 - слой доступа к данным (классы EFRepository, EFContext)
 - контроллер (класс ApiController).
При необходимости можно добавить слой бизнес-логики через который контроллер будет взаимодействовать с репозиторием.
- Управление зависимостями реализовано с помощью Autofac. 
- Репозиторий построен на ORM Microsoft Entity Framework Core.
- Настройка маппинга моделей в БД выполнена через Fluent API. При первом создании БД автоматически выполняется начальная загрузка тестовых данных.

3. Юнит тесты подготовлены в среде xUnit и выполняются на реальной БД (без использования MOQ-объектов).

3. Тестирование RESTapi выполнялось вручнаю утилитой POSTMAN: https://www.getpostman.com/
